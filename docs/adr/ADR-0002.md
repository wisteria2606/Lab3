# ADR-0002: Внедрение инструментов DevSecOps в CI/CD

## Контекст
Для обеспечения безопасности Python-приложения необходимо внедрить автоматические проверки кода и зависимостей на ранних этапах разработки (Shift-Left Security).

## Решение
Мы решили использовать **GitHub Actions** как платформу CI/CD и интегрировать следующие инструменты:

1.  **Gitleaks** — для предотвращения утечки секретов (API ключи, пароли) в репозиторий.
2.  **Semgrep** — как быстрый и легкий SAST-инструмент для поиска уязвимостей в коде Python.
3.  **Trivy** — универсальный сканер для поиска уязвимостей (CVE) в зависимостях (SCA) и Docker-образах.
4.  **Pytest** — для автоматического модульного тестирования функционала.

## Последствия
**Положительные:**
- Автоматическое выявление уязвимостей до деплоя.
- Стандартизация процесса сборки через Docker.
- Генерация SBOM для отслеживания цепочки поставок ПО.

**Отрицательные:**
- Время прохождения пайплайна увеличилось (из-за сканирования).
- Требуется регулярное обновление баз уязвимостей Trivy/Semgrep.
