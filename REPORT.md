# Отчет по лабораторной работе №3: DevSecOps

## Репозиторий
- **GitHub Repo:** https://github.com/wisteria2606/Lab3
- **Ветка:** main

## Результаты CI/CD
Был настроен автоматический пайплайн (Workflow) в GitHub Actions, включающий следующие этапы:

1. **Build and Test (Docker):**
   - Успешная сборка Docker-образа приложения.
   - Запуск unit-тестов (`pytest`) внутри контейнера.
   - Статус: ✅ Success.

2. **Security Checks (DevSecOps):**
   - **Gitleaks:** Поиск секретов и ключей в коде. (Использован официальный Action v2).
   - **Semgrep (SAST):** Статический анализ кода на уязвимости (OWASP Top 10).
   - **Trivy FS:** Сканирование файловой системы и зависимостей.
   - **Trivy Image:** Сканирование собранного Docker-образа на уязвимости ОС и пакетов.
   - **SBOM:** Генерация Software Bill of Materials в формате CycloneDX.

## Артефакты
В результате успешного прогона пайплайна были сформированы следующие отчеты (доступны во вкладке Actions -> Artifacts):
- `sbom` (файл `sbom.cdx.json`) — список всех компонентов ПО.

## Выводы
В ходе работы был внедрен подход DevSecOps. Проверки безопасности интегрированы непосредственно в процесс сборки (CI). Любой коммит теперь автоматически проверяется на наличие секретов, ошибок в коде и уязвимых библиотек, что предотвращает попадание небезопасного кода в продакшн.
